# Post-Quantum Cryptography Engine

The Sovereign AI node infrastructure backing the Orda blockchain requires future-roof financial transaction safety. For this reason, Qazaq IR natively incorporates the latest **NIST FIPS 204 (ML-DSA-44)** Post-Quantum Cryptography algorithms directly into its `orda_pqc` core execution engine.

## Zero Hidden Context Philosophy

A significant architectural hurdle when applying cryptographic signatures in deterministic LLM code pipelines is stateful key resolution. If a semantic pipeline generated by an LLM simply resolves to `SignTransaction`, a traditional compiler must arbitrarily search an unseen keystore or assume system context. This "Hidden Context" violates Qazaq IR's strictly deterministic O(1) constraint.

To solve this, the Suffix Morphemes of the Qazaq Lexer are structurally typed to enforce explicit Context injection directly from the Semantic Router payload.

### The Explicit Payload

When the LLM intends to sign data, it is mathematically forced to explicitly declare *which* alias or node key it intends to use:

```json
"morphs": [
    "AllocHeap", 
    { "SignWithMLDSA": "validator_primary_key" }, 
    "WriteToTarget"
]
```

### LLVM IR Emission

Because the context is explicit in the Suffix, the code generator can emit raw LLVM IR without relying on C++ bindings or guessing pointers. The explicit String key alias is carried through compilation into the executed binary bounds:

```llvm
%sig_ptr = call i8* @orda_pqc_mldsa_sign(i8* %transaction_ptr, i8* @validator_primary_key)
```

By ensuring that context is inherently explicit, the O(1) engine eliminates arbitrary injection points while supporting state-of-the-art cryptography.
